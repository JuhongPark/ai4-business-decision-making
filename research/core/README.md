## What Belongs Here

- foundational planning and scope documents
- literature review and reading notes
- case selection, domain analysis, and scenario design
- framework and taxonomy materials
- draft building blocks that feed the main outputs

## Naming Pattern

- Use descriptive filenames when a folder may contain multiple plans, summaries, or reviews over time.
- Keep cross-cutting reference documents here even if later deliverables cite them heavily.
- Treat `planning/`, `framework/`, and `taxonomy/` as the most canonical navigation points inside `core/`.
