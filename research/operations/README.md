## What Belongs Here

- application prep notes, answer drafts, and submission checklists
- roadmap and sequencing documents
- revision-stage synthesis, memos, and reviewer outputs
- expansion logic and refocus planning

## Language Rule

- All tracked documents in `operations/` must remain in English.
- The bilingual legacy exception does not apply in this zone.

## Boundary Rule

- If a document is mainly about process, iteration, or coordination, keep it here.
- If the document becomes a reusable analytical asset, move or duplicate the stable version into `core/`.
- Treat summary and plan documents as canonical here; detailed review files are supporting process evidence.
