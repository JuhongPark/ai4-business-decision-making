## Zone Map

### `core/`

Use this area for the stable analytical backbone of the project.

- `planning/`: scoping, timeline, and overall research-plan documents
- `questions/`: core research-question framing
- `literature/`: source tracker and reading notes
- `cases/`: case selection, mapping, and cross-case comparison
- `framework/`: evaluation framework and summary
- `drafting/`: section-level draft writing
- `evidence/`: evidence-strength and claim-discipline support
- `domain-analysis/`: cross-domain comparison and representative cases
- `scenario-design/`: scenario methodology and dataset shortlist
- `scenario-packs/`: domain-specific scenario packs
- `taxonomy/`: decision taxonomy, rules, and scoring tools

### `delivery/`

Use this area when you want polished or reader-facing outputs.

- `output/`: main report, executive summary, diagrams, and briefing assets
- `publication-prep/`: publication support tables and evidence registers
- `submission-ready/`: freeze, checklist, final validation, and claims register
- `validation/`: earlier validation pass materials kept for traceability

### `operations/`

Use this area for revision, application, and planning materials that support the core content.

- `applications/`: application prep documents, answer drafts, and submission notes
- `thinktank-program/`: roadmap and report architecture planning
- `revision/`: revision summaries, governance memo, and review-stage synthesis
- `expansion/`: expansion logic, evidence-tier method, and refocus planning

### `extensions/`

Use this area for follow-on workstreams that extend the main thesis.

- `three-part-study/`: workflow, infrastructure, and organization extension
- `adaptive-governance/`: governance timing, market, and implementation extension
- `information-structure/`: information shocks, consulting persistence, and AI-era decision structure
- `business-alignment/`: business-context alignment and operating-model implications
